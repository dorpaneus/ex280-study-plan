## 📖 Theory

### 1. The Storage Architecture
OpenShift uses a decoupled architecture for storage:
* **StorageClass (SC):** Defines the "provisioner" (the underlying driver that creates the storage) and parameters (e.g., SSD vs HDD).
* **PersistentVolume (PV):** A piece of actual storage in the cluster (like a physical disk).
* **PersistentVolumeClaim (PVC):** A request for storage by a user. The cluster matches the PVC to a suitable PV based on size, access mode, and storage class.

### 2. Access Modes
Understand the critical difference between these modes for the exam:
* `ReadWriteOnce` (RWO): Mounted as read-write by a single node. (Common for block storage).
* `ReadOnlyMany` (ROX): Mounted as read-only by many nodes.
* `ReadWriteMany` (RWX): Mounted as read-write by many nodes (requires specific shared storage like NFS or Ceph/OCS).

### 3. StatefulSets vs. Deployments
Unlike `Deployments`, `StatefulSets` provide:
* Stable, unique network identifiers.
* Stable, persistent storage across pod rescheduling.
* Ordered, graceful deployment and scaling.
* Mandatory `volumeClaimTemplates` for automated PV generation per replica.

## 🧪 Labs

### Lab 10.1: Dynamic Provisioning

**Step 1. Explore:** List available storage classes in your cluster.
`oc get sc`

**Step 2. Create:** Write and apply a PVC manifest. Create a file named `pvc.yaml`:

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: exam-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 2Gi
  storageClassName: standard

Apply it:
`oc apply -f pvc.yaml`

**Step 3. Verify:** Check the binding status (it should move from Pending to Bound):
`oc get pvc exam-pvc`


### Lab 10.2: StatefulSet Deployment

**Step 1. Deploy:** Create a StatefulSet that uses a `volumeClaimTemplate` to automatically request storage. Create `statefulset.yaml`:

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: web-store
spec:
  serviceName: "nginx"
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: registry.redhat.io/rhel8/nginx-120
        volumeMounts:
        - name: www-data
          mountPath: /usr/share/nginx/html
  volumeClaimTemplates:
  - metadata:
      name: www-data
    spec:
      accessModes: [ "ReadWriteOnce" ]
      resources:
        requests:
          storage: 1Gi

Apply it:
`oc apply -f statefulset.yaml`

**Step 2. Test Persistence:** Delete one of the pods and verify that when it restarts, it automatically re-attaches to the exact same PV.
`oc delete pod web-store-0`
`oc get pods -w`
`oc get pvc`
